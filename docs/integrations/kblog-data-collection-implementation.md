## Kblog Data Collection Integration

This guide documents how to reuse the Pathfinders Google Sheets workflow for Kblog so that contact messages and newsletter signups flow into the same spreadsheet.

---

### 1. Create the Google Sheet
- Make a new Google Sheet named something like `Kblog Engagement`.
- Add two tabs with these exact titles:
  - `Contact Messages`
  - `Newsletter Signups`
- Insert the headers below starting on row 1.

```text
Contact Messages:
Timestamp | Session ID | Source Page | Name | Email | Organization | Role | Subject | Message | Device Info (JSON) | IP Address | User Agent | Status

Newsletter Signups:
Timestamp | Session ID | Source Page | Email | Campaign Tag | Consent | Device Info (JSON) | IP Address | User Agent | Status
```

---

### 2. Deploy the Google Apps Script
1. Open **Extensions > Apps Script** from the spreadsheet.
2. Create a script file and paste the adjusted version of `@pathfinders/docs/scripts/google-apps-script.js`.
   - Replace the existing `switch(stepType)` with a check on `data.dataType` (`contact` or `newsletter`) and map to the new sheet/tab names above.
   - Ensure arrays are converted to strings and `deviceInfo` stays as `JSON.stringify`.
3. Save and deploy the project as a **Web app**:
   - Execute as: `Me`
   - Who has access: `Anyone`
4. Copy the deployment URL; you will reference it in Kblog’s environment variables.

---

### 3. Configure Environment Variables
- Add a `.env` entry for the deployment URL, for example:

```env
GS_DATA_PIPELINE_URL="https://script.google.com/macros/s/XXXXXXXXXXXX/exec"
```

- Expose it to the runtime where `@server.js` can read it (e.g., via `process.env.GS_DATA_PIPELINE_URL`).

---

### 4. Backend API Updates
1. Create a helper modeled on `submitToGoogleSheets` from `@pathfinders/lib/dataSubmission.ts`. Place it in a new module such as `@lib/googleSheetsClient.js` inside the Kblog project.
2. In `@server.js`, add two POST routes:
   - `/api/contact/submit` – validates the contact form payload, augments it with metadata (timestamp, sessionId, device info), and forwards it to the helper with `dataType: 'contact'`.
   - Update `/api/newsletter/subscribe` to call the helper with `dataType: 'newsletter'` instead of writing to `newsletter.json`. Retain the JSON write only as a fallback if required.
3. Normalize arrays to semicolon-separated strings and stringify nested objects before sending the request (match the transformation in `@pathfinders/app/api/data-submission/route.ts`).
4. Log success/failure and surface meaningful HTTP responses to the clients.

---

### 5. Frontend Form Wiring
1. Create `@assets/js/device-info.js` that mirrors `getDeviceInfo()` and `generateSessionId()` from `@pathfinders/lib/deviceInfo.ts`.
2. Contact form (`@contact.html`):
   - Link a new script (e.g., `@assets/js/contact.js`).
   - On submit, prevent default, read the form fields, call the backend route, and show success/error messages.
   - Include metadata (`sessionId`, `deviceInfo`, `pageUrl`) in the request body.
3. Newsletter forms (`@assets/js/newsletter.js`):
   - Import the new device/session utilities.
  - Replace the current simulated storage with a POST to `/api/newsletter/subscribe`.
   - Add graceful fallback if the network request fails (e.g., retry or local queue).

---

### 6. Testing & Verification
- Submit a contact form entry and confirm a row appears in the `Contact Messages` tab with all expected values.
- Submit newsletter signups from multiple pages and verify the `Newsletter Signups` tab records the source page/component.
- Inspect server logs for errors and ensure non-200 responses return actionable messages.

---

### 7. Maintenance Notes
- Rotate the Apps Script deployment URL if redeployed; update the environment variable accordingly.
- If more capture points are added (e.g., webinar registrations), extend the Apps Script switch with new `dataType` handlers and add new tabs.
- Document any future schema changes in this file and the deployment guide (`@docs/DEPLOYMENT_GUIDE.md`).
